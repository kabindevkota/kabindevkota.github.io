Objective 1 · Probability & Inference

Probability as the Foundation of Statistical Modeling

Diamond Project · Homework 1 (Auto & Boston SLR)

Every modeling decision in the diamond project rested on probability. The three-way train / validation / test split (34,505 / 8,629 / 10,786 rows) was designed so the test set was touched exactly once — a direct application of the assumption that test observations are independent draws from the same distribution.

In Homework 1, I fit a simple linear regression of mpg on horsepower (slope p-value = 7.03e-81) and computed 95% confidence and prediction intervals for horsepower = 98.

The prediction interval (14.81–34.12) is wider than the confidence interval (23.97–24.96) because it reflects the additional irreducible randomness of a single new observation, beyond uncertainty about the mean response.

The final diamond model's train R² of 0.998 and test R² of 0.997 — nearly identical — confirm the three-way split produced an honest, unbiased estimate with no data leakage.

Objective 2 · Generalized Linear Models

Applying the Appropriate GLM for the Data Context

PFI Education Survey Project · Homework 4 (Nonvoters)

The PFI project required three different GLMs within one dataset, each justified by the structure of the outcome. Binary logistic for high_achiever (0/1). Multinomial logistic for four-category grades. Poisson for FSFREQ (participation count) — but the dispersion ratio (residual deviance/df = 5.12, variance-to-mean ratio = 12.2) revealed severe overdispersion, so Negative Binomial was used instead, cutting AIC from 104,068 to 69,124. LDA and QDA were compared on five continuous predictors.

FSFREQ: mean = 7.02, variance = 85.4 → variance/mean ratio = 12.2. Standard Poisson assumes ratio = 1. This single diagnostic motivated switching models entirely.

Objective 3 · Model Selection

Demonstrating Model Selection Among Candidate Models

Diamond Project (Best Subset + Validation Set) · Homework 3 (Backward Elimination)

With 11 candidate predictors (including squared dimension terms x², y², z²), exhaustive best subset selection via regsubsets(nvmax=25) evaluated every possible model size up to 25 predictors. BIC, Cp, and adjusted R² gave conflicting signals.

Rather than trusting any single criterion, I used the held-out validation set — looping over all 25 sizes, computing prediction MSE via model.matrix() multiplication. The 5-predictor model (table, x, z, x², z²) had the lowest validation MSE (≈0.00069). The coefficient output from the report: intercept = 0.618, table = 0.00319, x = −0.701, z = 0.463, x² = 0.0746, z² = −0.0131.

Raw polynomial terms were replaced with poly(x,2) (orthogonal) to eliminate collinearity. GVIF values: table = 1.14, poly(x,2) = 66.5 (2 df → GVIF^(1/2df) = 2.86, acceptable).

# Validation set MSE loop (diamond project)
for (i in 1:n_models) {
  coef_i <- coef(regfit_full, id=i)
  pred_i <- val_matrix[,names(coef_i)] %*% coef_i
  val_errors[i] <- mean((val$carat - pred_i)^2)
}

Objective 4 · Communication

Expressing Statistical Results to a General Audience

PFI Education Survey Project · Homework 2 (Carseats MLR)

The PFI project opened with a Summary of Insights written for a parent or administrator, not a statistician. Key translation practice: log-odds → odds ratios → plain language.

Instead of "β = 1.26 for Democrat," I wrote: "Democrats have approximately 3.5× the odds of being a consistent voter compared to the Other group, after controlling for demographics."

In HW2, the prediction interval was explained in context: "The PI (679–1,364 units) is wider than the CI (978–1,064) because it reflects the additional uncertainty of predicting a single new store, not the average across all stores with those characteristics."

"A number is almost never a sufficient answer." Every coefficient, metric, and interval across all assignments was accompanied by a written interpretation in plain English.

Objective 5 · Programming in R

Using Software to Fit and Assess Statistical Models

Diamond Project · PFI Project · All Homeworks

Every assignment used R with tidymodels: the recipe → workflow → fit → augment → assess pipeline applied consistently whether the model was a simple linear regression or a multinomial classifier. New packages learned: discrim, poissonreg, themis, leaps, gghighlight, car.

10-fold cross-validation via vfold_cv() + fit_resamples() confirmed logistic model stability: CV AUC = 0.726 vs test AUC = 0.743 — a gap of 0.017, well within normal sampling variation.

Diagnostic plots became a genuine habit. In the diamond project, residuals vs fitted confirmed curvature requiring polynomial terms. VIF analysis motivated switching from raw x² to poly(x,2). These were not formalities — they shaped actual modeling decisions.

# Diamond project — full pipeline
final_model <- linear_reg() |> set_engine("lm") |>
  fit(carat ~ table + poly(x,2) + poly(z,2), data=diamonds_train)

augment(final_model, new_data=diamonds_test) |>
  metric_set(rmse,rsq,mae)(truth=carat, estimate=.pred)
