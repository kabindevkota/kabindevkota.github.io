Objective 1 · Probability & Inference

Probability as the Foundation of Statistical Modeling

Diamond Project · Homework 1 (Auto & Boston SLR)
The statistical foundation
The idea
Every regression model is a probability statement. We're finding the coefficients that make our observed data as probable as possible — this process is called Maximum Likelihood Estimation (MLE). For linear regression, MLE is equivalent to minimizing squared residuals, but the probabilistic framing explains why that works.
In practice
Each coefficient β comes with a standard error (uncertainty), a t-statistic (signal-to-noise), and a p-value (probability of seeing this result if β were really zero). A 95% confidence interval is: estimate ± 1.96 × std.error. If it excludes zero, the predictor carries genuine signal.
Key distinction
A confidence interval answers: "Where is the true average response?" A prediction interval answers: "Where will the next individual observation fall?" The prediction interval is always wider — it must account for individual variation on top of estimation uncertainty.
MLE mechanics
The likelihood function for linear regression is: L(β) = ∏ P(yᵢ | xᵢ, β). MLE finds the β that maximizes this — equivalent to minimizing squared residuals. Standard errors come from the Fisher Information Matrix (the curvature of the log-likelihood at the peak). A steeper peak = more information = smaller standard errors = more precise estimates.
CI formula
Every 95% confidence interval in this portfolio is: estimate ± 1.96 × std.error. The 1.96 comes from the normal distribution — for large samples, MLE estimates are approximately normal. Wald tests use the same standard errors to compute t-statistics and p-values for each coefficient.

In Homework 1, I fit mpg ~ horsepower on the Auto dataset. The slope's p-value of 7.03e-81 is effectively zero — if horsepower had no real effect, the chance of seeing a relationship this strong by luck alone is vanishingly small. The slope (−0.158) means each additional unit of horsepower is associated with 0.158 fewer miles per gallon.

I computed 95% confidence and prediction intervals for horsepower = 98. The CI (23.97–24.96) is tight — about the average mpg. The PI (14.81–34.12) is nearly five times wider — accounting for how much individual cars vary around that average.

The PI is wide not because our model is uncertain, but because individual cars vary a lot even with the same horsepower. Some run hot, some run cold. That's irreducible variance — and the prediction interval honestly captures it.

The diamond project extended this to a three-way split (34,505 train / 8,629 validation / 10,786 test rows). The test set was touched exactly once, at the very end. This is probabilistic discipline: peeking at test performance during model-building would inflate our reported R² by overfitting to test-set noise.

Objective 2 · Generalized Linear Models

Applying the Appropriate GLM for the Data Context

PFI Education Survey Project · Homework 4 (Nonvoters)
Why the model must match the outcome
Binary (0/1)
Logistic regression. The outcome is either "high achiever" or not. A standard linear regression would give predicted probabilities outside [0,1], which makes no sense. Logistic regression uses the logit link — log(p / 1−p) = β₀ + β₁X₁ + … — keeping all predictions between 0 and 1 and producing odds ratios as interpretable outputs.
4 categories
Multinomial logistic. Grades come in four unordered groups (Mostly A / B / C / D). Binary logistic can't handle this. Multinomial fits K−1 simultaneous equations, each comparing one class to a reference: log(P(B) / P(A)) = …, log(P(C) / P(A)) = …, etc.
Count data
Poisson → Negative Binomial. School participation (FSFREQ) is a non-negative integer count. Poisson is the natural starting point but assumes mean = variance. When the data shows variance/mean = 12.2, Poisson's standard errors are severely underestimated — everything looks falsely significant. Negative Binomial adds a dispersion parameter θ to handle this.
Numeric only
LDA / QDA. These methods model each class as a multivariate normal distribution and find the boundary that best separates them. They require continuous predictors — so we used five: school involvement score, home engagement score, income, dinner frequency, and number of siblings.

The PFI project required choosing a different GLM for each of three outcomes — binary, four-class, and count — all from the same dataset. The decision was never automatic; it came from examining what kind of thing we were trying to predict.

For the count outcome (school participation), Poisson was the natural first choice. But two diagnostics flagged a problem: the variance-to-mean ratio was 12.2 (Poisson requires 1.0), and the residual deviance/df ratio was 5.12 (again, should be near 1). This wasn't a minor issue — it means Poisson was severely underestimating uncertainty, making weak predictors appear highly significant.

Switching from Poisson to Negative Binomial reduced the AIC from 104,068 to 69,124 — an improvement of ~35,000 points. In model comparison, a difference of 10 AIC points is considered decisive. This difference was 35,000.

For the binary outcome (high achiever), logistic regression (AUC = 0.743) substantially outperformed both LDA (0.651) and QDA (0.650). The gap exists because logistic regression can incorporate categorical predictors — disability status, race, parent education — that LDA and QDA cannot use in their standard formulation.

Objective 3 · Model Selection

Demonstrating Model Selection Among Candidate Models

Diamond Project (Best Subset + Validation Set) · Homework 3 (Backward Elimination)
The model selection problem
The risk
With 11 candidate predictors, there are 2¹¹ = 2,048 possible models. Picking the one with the highest training R² will always find something that looks impressive — but it's just memorizing the training data. That model will disappoint on new observations it hasn't seen before.
The solution
Fit models on training data. Choose between them using a separate validation set that was never used during fitting. The validation MSE is an honest estimate of real-world performance. Only after selecting the model do we evaluate on the test set — once, at the very end.
Occam's Razor
When a simpler model achieves the same accuracy as a complex one, prefer the simpler one. It's easier to explain, less prone to overfitting, and more trustworthy. In the diamond project, once validation MSE plateaued at 5 predictors, there was no reason to add more — complexity without benefit is just noise.

With 11 candidate predictors (including squared dimension terms x², y², z²), exhaustive best subset selection via regsubsets(nvmax=25) evaluated every possible model at every size. Four criteria — RSS, adj R², Cp, BIC — pointed to different "best" sizes, with BIC suggesting ~13 predictors and adj R² plateauing at ~8.

Rather than arbitrarily trusting one criterion, I looped over all 25 model sizes, applied each model's coefficients to the held-out validation set via matrix multiplication, and computed prediction MSE. The 5-predictor model (table, x, z, x², z²) won — validation MSE ≈ 0.00069, the sharpest drop in the curve, with flat improvement thereafter.

After selection, raw polynomial terms were replaced with orthogonal polynomials via poly(x,2). Raw x and x² are correlated by construction (VIF inflates), but orthogonal versions capture the same curve with independent components. The VIF check confirmed: poly(x,2) GVIF^(1/2df) = 2.86, well within acceptable range.

# Loop: evaluate each model size on validation set
for (i in 1:n_models) {
  coef_i <- coef(regfit_full, id=i)
  pred_i <- val_matrix[,names(coef_i)] %*% coef_i
  val_errors[i] <- mean((val$carat - pred_i)^2)
}
best_size <- which.min(val_errors) # → 5

# Use orthogonal poly to resolve collinearity
fit(carat ~ table + poly(x,2) + poly(z,2), data=diamonds_train)

The PFI project opened with a plain-language summary for a parent or school administrator — before any model output appeared. The goal was to answer "what did we find?" for someone who will never read a coefficient table.

Key translation rule: log-odds are never reported directly. Every coefficient was converted to an odds ratio, then to a plain statement. Instead of "β = 1.43 for DSBLTY," the report says: "Children without a disability are more than four times as likely to earn mostly A's."

In Homework 2, the prediction interval for a hypothetical new store was not just reported as numbers: "The interval 679–1,364 units is wider than the confidence interval 978–1,064 because predicting a single store's sales involves more uncertainty than estimating the average across all stores with those characteristics."

"A number is almost never a sufficient answer." — Every coefficient, metric, and interval across all assignments was accompanied by a sentence explaining what it means in the context of the real problem being studied.

The interaction term in HW2 was explained in plain terms: "The negative effect of price on sales is slightly stronger in good shelf placements — highly visible products may attract more price-sensitive shoppers." The interaction was then noted as non-significant, which was also communicated honestly rather than ignored.

What this means for schools and policymakers
For Schools

Focus parent engagement programs on families where disability and lower parental education are both present — these children face compounding disadvantages that show up clearly in the model. Increasing one school activity for these families is associated with a 60% boost in high-achievement odds.

For Policymakers

Parent education is a strong predictor but not one schools can change directly. The actionable lever is school involvement: activities, volunteering, and family-school communication. These are in scope for intervention and show consistent, significant effects across all model types.

Model Limitations

This model can rank students by predicted risk and identify associated patterns. It cannot establish causation, predict any individual child's outcome with certainty, or account for unmeasured factors like school quality or teacher experience. AUC = 0.743 means 1-in-4 rankings are still wrong.

What the Model Cannot Do

Predict with certainty which specific child will succeed · Fully separate correlation from causation · Account for unmeasured variables · Replace human judgment about a child's circumstances · Guarantee the same results on a different survey year or population.

Objective 5 · Programming in R

Using Software to Fit and Assess Statistical Models

Diamond Project · PFI Project · All Homeworks

Every assignment used R with the tidymodels ecosystem. The core pattern — recipe → workflow → fit → augment → assess — stayed consistent whether the model was a simple linear regression or a multinomial classifier with upsampling. This consistency makes code easier to read, compare, and extend later.

New packages learned: discrim (LDA/QDA via MASS), poissonreg (Poisson in tidymodels), themis (upsampling imbalanced classes), leaps (best subset selection), gghighlight (outlier highlighting), car (VIF computation).

10-fold cross-validation via vfold_cv() + fit_resamples() confirmed logistic model stability: CV AUC = 0.726 vs test AUC = 0.743. The gap of 0.017 is within normal sampling variation — no overfitting.

Diagnostic plots were real decision tools. In the diamond project, the residuals vs fitted plot showing curvature confirmed polynomial terms were necessary. The VIF check motivated switching from raw x² to poly(x,2). These were not formalities — they shaped the final model.
